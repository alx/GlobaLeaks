Welcome to the Interactive Demo

In this demo you will experience the very basic workflow of GlobaLeaks:

  • A GlobaLeaks node administrator setups up the targets list. These will be the receivers of the submitted material.
  • The Whistleblower submits the material and gets back a 10 digit receipt.
  • The Target receives the submitted material as a tulip that will expire after a certain amount of downloads
  • The Whistleblower using the receipt checks the status of his submission

The basic functionalities that this prototype is missing are: comments, tulip password authentication and much more, to have a full list visit our blueprints page

Warning: GlobaLeaks is by no means ready for deployment in a real world scenario as it is still just a prototype. We just make software and we will never run a node for appropriated responsability segregation. Upload functionality it's disabled for the prototype demo.
